# Data Model: Discogs Conversational Analytics Agent — V1

**Feature**: [spec.md](./spec.md) | **Plan**: [plan.md](./plan.md)
**Audience**: implementer of `agent/src/discogs_agent/persistence/`
and the LangGraph nodes — what entities exist, what their fields
mean, how they relate, and how they transition.

There are **two storage layers**:

1. **Postgres** — the durable trace store. Six tables. Modeled
   in SQLAlchemy 2.x. Migrations under
   `agent/src/discogs_agent/persistence/migrations/`.
2. **The in-memory graph state** (`AgentState` TypedDict) —
   per-request, never persisted as a row, but its derived
   fields are projected into the Postgres tables as the run
   progresses.

Plus one read-only contract:

3. **`SchemaContext`** — what the agent observes about the
   published DuckDB at startup. Not a row anywhere; built once
   by `dataset_schema_reader` and held in module-level cache.

The DDL is in [`contracts/postgres-schema.md`](./contracts/postgres-schema.md).
This file documents the **logical** model.

---

## 1. Persisted entities (Postgres)

### 1.1 `Thread` (`agent_threads`)

A logical grouping of related runs that share a conversation
context.

| Field | Type | Nullable | Notes |
|-------|------|----------|-------|
| `thread_id` | UUID | NO (PK) | Generated by the API on create; reused across runs. |
| `created_at` | TIMESTAMPTZ | NO | `DEFAULT now()`. |
| `updated_at` | TIMESTAMPTZ | NO | Updated on every new run; trigger or app-side. |
| `status` | VARCHAR(32) | NO | Enum: `active`, `archived`. V1 only writes `active`. |
| `metadata_json` | JSONB | NO (default `'{}'::jsonb`) | Free-form. V1 reserves the key `carryover_summary` for an aggregated representation of prior runs (optional). |

**Validation rules**:
- A thread is created on the first `/query` call that doesn't
  pass a `thread_id`.
- A `/query` that passes an unknown `thread_id` MUST return
  `404` (FR-032 implied — the thread must exist or be created
  fresh).

**State transitions**:
- `(none) → active` on first run.
- `active → archived` reserved for future work; V1 never
  triggers this.

### 1.2 `Run` (`agent_runs`)

A single user-question → response cycle. Belongs to exactly
one thread.

| Field | Type | Nullable | Notes |
|-------|------|----------|-------|
| `run_id` | UUID | NO (PK) | Returned in `/query` response. |
| `thread_id` | UUID | NO (FK → `agent_threads`) | |
| `user_query` | TEXT | NO | The natural-language question, verbatim. |
| `complexity` | VARCHAR(32) | YES | Enum: `simple`, `complex`, `unsupported`, `clarification_needed`. NULL until router runs. |
| `selected_model` | VARCHAR(64) | YES | The actual model name (e.g., `gpt-4o-mini`). NULL for `unsupported` / `clarification_needed` (no codegen). |
| `status` | VARCHAR(32) | NO | Enum: `running`, `succeeded`, `failed_safety`, `failed_validation`, `failed_unsupported`, `failed_clarification_needed`, `failed_internal`. Initial value `running`; updated to a terminal value before the response is returned. |
| `started_at` | TIMESTAMPTZ | NO | `DEFAULT now()`. |
| `finished_at` | TIMESTAMPTZ | YES | NULL while running. |
| `latency_ms` | INTEGER | YES | NULL while running. Computed at finish. |
| `final_response` | TEXT | YES | The user-facing reply. NULL while running. |
| `generated_sql` | TEXT | YES | NULL for `unsupported` / `clarification_needed` and for runs that errored before code execution. |
| `metadata_json` | JSONB | NO (default `'{}'::jsonb`) | Reserved keys: `carryover` (the prior-runs preamble injected into `query_understanding`, FR-032), `route_rationale` (router's natural-language explanation), `retry_count` (final retry counter at finish). |

**Validation rules**:
- `complexity` set ⇒ `selected_model` set, **except** when
  `complexity ∈ {unsupported, clarification_needed}`.
- `status = succeeded` ⇒ at least one `agent_artifact` row
  exists for this `run_id`.
- `status ∈ failed_*` ⇒ at least one `agent_error` row exists
  (informational; not enforced by FK).
- `latency_ms = (finished_at - started_at) in ms`. Computed
  app-side at finish.

**State transitions** (for `status`):

```text
running
  ├── succeeded                       (chart validated, response synthesized)
  ├── failed_unsupported              (router classified as unsupported)
  ├── failed_clarification_needed     (router classified as clarification_needed)
  ├── failed_safety                   (safety retries exhausted)
  ├── failed_validation               (validator retries exhausted)
  └── failed_internal                 (an unexpected exception escaped a node)
```

Once a terminal state is set, the row is immutable (V1 has no
"resume run" capability — only "resume thread").

### 1.3 `ToolCall` (`agent_tool_calls`)

A single invocation of a named tool by a named graph node
within a run.

| Field | Type | Nullable | Notes |
|-------|------|----------|-------|
| `tool_call_id` | UUID | NO (PK) | |
| `run_id` | UUID | NO (FK → `agent_runs`) | |
| `node_name` | VARCHAR(64) | NO | One of the eight node names. |
| `tool_name` | VARCHAR(64) | NO | One of the seven tool names. |
| `input_json` | JSONB | NO | Tool input, redacted of secrets. |
| `output_json` | JSONB | YES | Tool output. NULL on error. |
| `status` | VARCHAR(32) | NO | Enum: `succeeded`, `failed`. |
| `latency_ms` | INTEGER | NO | |
| `error_message` | TEXT | YES | NULL on success. |
| `created_at` | TIMESTAMPTZ | NO | `DEFAULT now()`. |

**Validation rules**:
- `(node_name, tool_name)` MUST be in the allowlist defined
  in `contracts/tools.md` §3.
- `status = failed` ⇒ `output_json IS NULL`.
- `status = failed` ⇒ `error_message IS NOT NULL`.

**State transitions**: rows are append-only; no updates.

### 1.4 `ModelUsage` (`agent_model_usage`)

A single LLM invocation within a run.

| Field | Type | Nullable | Notes |
|-------|------|----------|-------|
| `usage_id` | UUID | NO (PK) | |
| `run_id` | UUID | NO (FK → `agent_runs`) | |
| `node_name` | VARCHAR(64) | NO | One of the eight node names. |
| `model_name` | VARCHAR(64) | NO | The actual OpenAI model name. |
| `prompt_tokens` | INTEGER | NO | From OpenAI response `usage`. |
| `completion_tokens` | INTEGER | NO | From OpenAI response `usage`. |
| `total_tokens` | INTEGER | NO | `prompt + completion`. Stored explicitly for cheap aggregation. |
| `estimated_cost_usd` | NUMERIC(10,6) | YES | NULL when the model name isn't in the rate card; warning logged. |
| `latency_ms` | INTEGER | NO | API-call wall-clock. |
| `created_at` | TIMESTAMPTZ | NO | `DEFAULT now()`. |

**Validation rules**:
- `total_tokens = prompt_tokens + completion_tokens`.
- A run with at least one supported (`simple` / `complex`)
  classification SHOULD have ≥ 2 rows: one for the
  router/classifier call, one for code generation. The
  inspection API doesn't enforce this; it's a sanity check
  for tests.

**State transitions**: append-only.

---

### 1.5 `Artifact` (`agent_artifacts`)

A generated chart file produced by a run.

| Field | Type | Nullable | Notes |
|-------|------|----------|-------|
| `artifact_id` | UUID | NO (PK) | |
| `run_id` | UUID | NO (FK → `agent_runs`) | |
| `thread_id` | UUID | NO (FK → `agent_threads`) | Denormalized for browse-by-thread. |
| `artifact_type` | VARCHAR(32) | NO | V1: only `plotly_html`. Future: `csv`, `parquet`, `image_png`. |
| `path` | TEXT | NO | Filesystem path **inside the agent container**, e.g., `/app/artifacts/<thread_id>/<run_id>/chart.html`. |
| `metadata_json` | JSONB | NO (default `'{}'::jsonb`) | V1 keys: `bytes` (file size), `chart_type` (e.g., `bar`, `line`), `row_count` (rows in the underlying dataframe preview). |
| `created_at` | TIMESTAMPTZ | NO | `DEFAULT now()`. |

**Validation rules**:
- `path` MUST be inside `ARTIFACTS_DIR`. Enforced by the
  artifact-store tool before insert (path-traversal guard).
- `artifact_type = plotly_html` ⇒ `path` ends with `.html`.
- A successful run produces **exactly one** artifact in V1
  (FR-018). Multiple artifacts per run is future work
  (artifact gallery).

**State transitions**: append-only. Files on disk are
**not** deleted in V1 (artifact retention policy is future
work).

---

### 1.6 `Error` (`agent_errors`)

A recorded failure during a run.

| Field | Type | Nullable | Notes |
|-------|------|----------|-------|
| `error_id` | UUID | NO (PK) | |
| `run_id` | UUID | NO (FK → `agent_runs`) | |
| `node_name` | VARCHAR(64) | NO | The node that raised. |
| `error_type` | VARCHAR(64) | NO | Classified bucket: `safety_violation`, `sandbox_timeout`, `sandbox_exception`, `validation_failed`, `unexpected`. |
| `error_message` | TEXT | NO | Short, user-facing-ish summary. |
| `traceback` | TEXT | YES | Full traceback for `unexpected`. NULL for the classified buckets (safety/timeout/etc). End-user responses MUST NOT expose this; only admin/debug clients of the inspection API may see it. |
| `created_at` | TIMESTAMPTZ | NO | `DEFAULT now()`. |

**Validation rules**:
- `error_type = unexpected` ⇒ `traceback IS NOT NULL`.
- Other buckets MAY have `traceback = NULL`.

**State transitions**: append-only.

## 2. In-memory state (LangGraph)

### 2.1 `AgentState` (TypedDict)

The state struct that flows through the LangGraph. Lives only
for the duration of one request. Fields are progressively
populated as nodes run; nodes MUST NOT mutate fields owned by
earlier nodes.

```python
from typing import TypedDict, Any

class AgentState(TypedDict):
    # Identity (set by the API, before invoke)
    thread_id: str
    run_id: str
    user_query: str

    # Carry-over (set by query_understanding before LLM call)
    carryover_preamble: str | None      # the formatted prior-runs text
    carryover_turn_count: int           # how many prior runs included

    # Schema (set by load_schema)
    schema_context: dict[str, Any]      # see SchemaContext below

    # Routing (set by router)
    route: dict[str, Any] | None        # {complexity, selected_model, rationale}

    # Plan (set by query_understanding)
    query_plan: dict[str, Any] | None   # tables, dimensions, metrics, filters, chart_type, notes

    # Generation (set by code_generator)
    generated_code: str | None
    generated_sql: str | None           # extracted by safety checker; mirrored here for trace

    # Validation outputs
    safety_result: dict[str, Any] | None       # {allowed, violations}
    execution_result: dict[str, Any] | None    # {exit_code, stdout, stderr, result, duration_ms}
    validation_result: dict[str, Any] | None   # {valid, errors, should_retry}

    # Artifacts
    artifact_paths: list[str]
    dataframe_preview: list[dict[str, Any]]

    # Retry control
    retry_count: int
    max_retries: int

    # Trace accumulators (mirrored to Postgres after each node)
    errors: list[dict[str, Any]]
    model_usage: list[dict[str, Any]]
    tool_calls: list[dict[str, Any]]

    # Final
    final_response: str | None
```

**Field ownership**:

| Field | Set by | May read |
|-------|--------|----------|
| `thread_id`, `run_id`, `user_query` | API entry | all |
| `carryover_*` | `query_understanding` (before LLM) | `query_understanding` only |
| `schema_context` | `load_schema` | `router`, `query_understanding`, `code_generator`, `response_synthesizer` |
| `route` | `router` | downstream |
| `query_plan` | `query_understanding` | `code_generator`, `response_synthesizer` |
| `generated_code` | `code_generator` | `sql_safety_checker`, `sandbox_executor`, `response_synthesizer` |
| `generated_sql` | `sql_safety_checker` (extracted from code) | downstream |
| `safety_result` | `sql_safety_checker` | retry-decision edge |
| `execution_result` | `sandbox_executor` | `chart_validator`, `response_synthesizer` |
| `validation_result` | `chart_validator` | retry-decision edge |
| `artifact_paths`, `dataframe_preview` | `sandbox_executor` (mirrors `RESULT`) | `chart_validator`, `response_synthesizer` |
| `retry_count` | `code_generator` increments on each entry | retry-decision edges |
| `errors`, `model_usage`, `tool_calls` | every node may append; never read for control flow | trace projection |
| `final_response` | `response_synthesizer` | the API |

**Persistence projection**: at the end of every node, a trace
shim writes the new `tool_calls` / `model_usage` / `errors`
entries to Postgres (deltas only). At the end of the run, the
final `agent_runs` row is updated with `complexity`,
`selected_model`, `status`, `latency_ms`, `final_response`,
`generated_sql`, and `metadata_json`.

### 2.2 `SchemaContext` (built once at startup)

Read-only view of the published DuckDB. Built by
`dataset_schema_reader` on FastAPI startup; cached in a
module-level variable; refreshed only on process restart.
Stale-cache risk is acceptable in V1 (the ETL does not write to
the published DuckDB while the agent is up — it produces a new
file).

```python
class SchemaContext(TypedDict):
    tables: dict[str, list[dict[str, str]]]   # {table_name: [{name, type}, ...]}
    has_master_fact: bool
    duckdb_path: str
    captured_at: str                          # ISO timestamp
```

**Validation rules** (asserted at build time; raise on failure
so `/health` reports it):
- The four core tables/views MUST be present:
  `release_fact`, `release_unique_view`,
  `release_artist_bridge`, `release_label_bridge`.
- `master_fact` is optional; presence sets `has_master_fact`.
- No `stg_*` / `clean_*` / `release_format_summary` table is
  exposed in `tables` — the reader **filters them out** even
  if present in the file. (Defense-in-depth — the published
  DuckDB shouldn't contain them, but it's cheap to filter.)

## 3. Cross-cutting invariants

These hold across the whole model:

1. **One source of truth per fact.** A run's `final_response`
   lives in `agent_runs.final_response`, not also in
   `agent_artifacts.metadata_json`. The trace tables are
   denormalized only where browsing efficiency demands it
   (e.g., `agent_artifacts.thread_id`).
2. **Append-only trace.** `agent_tool_calls`,
   `agent_model_usage`, `agent_errors`, `agent_artifacts` are
   never updated after insert. `agent_threads.updated_at`
   and `agent_runs.{status, finished_at, latency_ms,
   final_response, generated_sql, metadata_json}` are the only
   mutable fields.
3. **Read-only DuckDB.** No code path opens the published
   DuckDB with `read_only=False`. Verified by an integration
   test that asserts byte-equality of the file before and
   after a documented batch of queries (SC-007).
4. **Allowlist enforcement happens before execution.** The
   `sql_safety_checker` runs before `sandbox_executor`, with no
   bypass. Verified by a graph-path test that injects a known
   bad SQL and asserts the sandbox is never invoked.
5. **No raw tracebacks leak to end-users.** `agent_errors.traceback`
   is exposed only via the admin/debug branch of the
   inspection API (controlled by an env flag, not by request
   shape). End-user `/query` responses NEVER contain
   `traceback` content. Verified by an integration test on a
   deliberate sandbox failure.
